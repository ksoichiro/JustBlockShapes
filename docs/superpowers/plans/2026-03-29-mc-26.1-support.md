# Minecraft 26.1 Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Minecraft 26.1 support with Gradle 9 upgrade, ForgeGradle 7.x migration, and new Fabric/NeoForge modules (Forge disabled due to upstream crash).

**Architecture:** MC 26.x+ uses a different build approach than 1.x: common module uses `net.neoforged.moddev` (NeoForm), Fabric uses `net.fabricmc.fabric-loom`, NeoForge uses `net.neoforged.moddev` — all without Architectury Loom/Plugin. Platform modules consume common code via explicit configurations (`commonJava`/`commonResources`) instead of Architectury's `namedElements`/`transformProduction`. Existing 1.21.x ForgeGradle migrates from 6.x to 7.x.

**Tech Stack:** Gradle 9.4.0, Fabric Loom 1.15.5, ModDevGradle 2.0.141, ForgeGradle 7.0.17 (local build), Java 25

**Reference:** JustCoordinates project at `../JustCoordinates/` implements this same approach.

### Task 4: Update multi-version-tasks.gradle for Gradle 9

**Files:**
- Modify: `gradle/multi-version-tasks.gradle`

- [ ] **Step 1: Replace `project.exec` with `ProcessBuilder`**

Gradle 9 deprecates `project.exec` in task actions. Replace all `project.exec { commandLine ... }` calls with a `runGradle` helper using `ProcessBuilder`, matching JustCoordinates pattern:

```groovy
def runGradle = { List args ->
    def cmd = ['./gradlew'] + args.collect { it.toString() }
    def proc = new ProcessBuilder(cmd as String[])
            .directory(rootDir)
            .inheritIO()
            .start()
    def exitCode = proc.waitFor()
    if (exitCode != 0) {
        throw new GradleException("Command failed (exit ${exitCode}): ${cmd.join(' ')}")
    }
}
```

- [ ] **Step 1: Create `fabric/26.1/build.gradle`**

Uses `net.fabricmc.fabric-loom` (not `dev.architectury.loom`). No Architectury shadow/transform. Consumes common module via `commonJava`/`commonResources` configurations.

```groovy
plugins {
    id 'net.fabricmc.fabric-loom'
}

configurations {
    commonJava {
        canBeResolved = true
    }
    commonResources {
        canBeResolved = true
    }
}

dependencies {
    minecraft "com.mojang:minecraft:${minecraft_version}"
    implementation "net.fabricmc:fabric-loader:${fabric_loader_version}"
    implementation "net.fabricmc.fabric-api:fabric-api:${fabric_api_version}"

    compileOnly project(':common')

    commonJava project(path: ':common', configuration: 'commonJava')
    commonResources project(path: ':common', configuration: 'commonResources')
}

sourceSets.main {
    java {
        srcDir rootProject.file('common/shared/src/main/java')
    }
}

tasks.named('compileJava') {
    dependsOn configurations.commonJava
    source configurations.commonJava
}

tasks.named('processResources') {
    dependsOn configurations.commonResources
    from configurations.commonResources

    filesMatching('fabric.mod.json') {
        expand(
            version: mod_version,
            minecraft_version: minecraft_version,
            fabric_loader_version: fabric_loader_version,
        )
    }
}

loom {
    runs {
        client {
            client()
            runDir("run")
        }
        server {
            server()
            runDir("run")
        }
        gameTest {
            server()
            runDir("run")
            vmArg "-Dfabric-api.gametest"
        }
    }
}

jar {
    archiveBaseName = rootProject.archives_name
    archiveVersion = "${rootProject.mod_version}+${minecraft_version}-fabric"
    archiveClassifier = null
}

sourcesJar {
    archiveBaseName = rootProject.archives_name
    archiveVersion = "${rootProject.mod_version}+${minecraft_version}-fabric"
}
```
