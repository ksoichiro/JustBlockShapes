## Uncommitted Changes (2026-04-04)

### architectury-plugin removal + commonJava/commonResources migration

`architectury-plugin` (all versions) is fundamentally incompatible with Gradle 9 (`ClassNotFoundException: net.fabricmc.loom.LoomGradleExtension`). Removed it and adopted direct source sharing pattern.

**Plugin migration:**
| Module | Before | After |
|--------|--------|-------|
| common (1.21.x) | `dev.architectury.loom` (implicit) | `fabric-loom 1.13.6` |
| fabric (1.21.x) | `dev.architectury.loom` + `com.gradleup.shadow` | `fabric-loom 1.13.6` |
| neoforge (1.21.x) | `dev.architectury.loom` (implicit) + `com.gradleup.shadow` | `net.neoforged.moddev` |
| forge (1.21.x) | `net.minecraftforge.gradle` (unchanged) | unchanged |

- `settings.gradle`: Replaced architectury plugins with `fabric-loom`, removed `maven.architectury.dev`
- `build.gradle` (root): Removed architectury-plugin, simplified to flat repo list + moddev declaration
- `gradle.properties`: Added `fabric_loom_version=1.13.6`
- All `props/*.properties` (1.21.1–1.21.11): Added `neoform_version`, temporarily disabled Forge

## Known Issues

### Door handle texture generation (NeoForge 26.1)
- `DoorHandleTextureGenerator` cannot read vanilla textures in NeoForge 26.1
- Classpath fallback (`getResourceAsStream`) fails: NeoForm/ModDev doesn't put vanilla textures on classpath
- `TextureProvider` via `FMLClientSetupEvent` fires AFTER resource loading — too late

### Fabric 26.1 — Fabric Loom + Gradle 9
- Fabric Loom 1.15.5 fails at project configuration with GSON 2.8.6 `IllegalAccessException`
- Fabric 26.1 disabled in `enabled_platforms`
- Note: Fabric Loom 1.13.6 (used for 1.21.x) works fine with Gradle 9

### Forge 1.21.x — ForgeGradle 7.x API changes
- `reobf`, `copyIdeResources`, `workingDirectory()` removed/changed in ForgeGradle 7.x
- All 1.21.x `enabled_platforms` temporarily set to `fabric,neoforge`
- Prerequisite: run `./scripts/build-forgegradle-local.sh` to publish patched ForgeGradle to mavenLocal

### Forge 26.1 — upstream crash
- Forge 62.0.x crashes at startup (FieldToMethodTransformer bug)
- Skeleton created but cannot be tested
